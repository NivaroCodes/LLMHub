Integration

Start in 3 lines

Pick your flavor. The gateway speaks fluent Python, OpenAI, and plain HTTP. 

from llmhub import LLMHub

hub = LLMHub(base_url="https://llmhub-production.up.railway.app")
response = hub.chat_sync("Explain this error")

print(response.answer)
print(f"via {response.provider} — ${response.cost_usd}")

from openai import OpenAI

client = OpenAI(
    base_url="https://llmhub-production.up.railway.app/v1",
    api_key="any"
)

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Hello"}]
)

# Send a chat request
curl -X POST https://llmhub-production.up.railway.app/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello!"}'

# Response:
{
  "answer": "Hello! How can I help?",
  "provider": "openrouter",
  "cost_usd": 0.00001155,
  "fallback_used": true
}

Installation

LLMHub installs from PyPI. Python 3.9+ is required. The gateway runs anywhere Python runs — your laptop, a Docker container, or Railway. 

$ pip install llmhub

Or, if you prefer Poetry:

$ poetry add llmhub

Quick Start

Three lines and you're routing requests. The hosted gateway at llmhub-production.up.railway.app is open for testing — for production, self-host. 

from llmhub import LLMHub

hub = LLMHub(base_url="https://llmhub-production.up.railway.app")
response = hub.chat_sync("What is the capital of France?")

print(response.answer)       # "Paris"
print(response.provider)     # "ollama" or "gemini" or...
print(response.cost_usd)     # 0.0 if local, < $0.001 otherwise

Environment Variables

Configure providers via environment variables. Only set the ones you want to use — LLMHub skips unconfigured providers gracefully.

Variable Default Description
OLLAMA_HOST http://localhost:11434 Local Ollama endpoint. Used as the primary provider.
OLLAMA_MODEL llama3.2 Default model name passed to Ollama.
GEMINI_API_KEY — Google AI Studio API key. Free tier is generous.
OPENROUTER_API_KEY — OpenRouter key. Cheapest cloud fallback.
OPENAI_API_KEY — OpenAI key. Last resort in the fallback chain.
REDIS_URL — If set, enables semantic caching.
LLMHUB_PORT 8000 HTTP port to bind on.

Run the Gateway

Once installed, you can run the gateway as a standalone HTTP server: 

$ llmhub serve --port 8000

Then point your existing OpenAI client at http://localhost:8000/v1 . That's it. Everything else works the same.
